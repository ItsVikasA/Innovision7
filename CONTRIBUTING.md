Thank you for your interest in contributing to InnoVision! We welcome contributions from developers of all skill levels. This guide will help you get started.

## üåü How to Contribute

### 1. Fork and Clone

1. Fork the repository by clicking the "Fork" button at the top right
2. Clone your forked repository:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Innovision7.git
   cd Innovision7
   ```
3. Add upstream remote:
   ```bash
   git remote add upstream https://github.com/ItsVikasA/Innovision7.git
   ```

### 2. Set Up Development Environment

1. Install Node.js 18+ from [nodejs.org](https://nodejs.org/)
2. Install dependencies:
   ```bash
   npm install
   ```
3. Set up environment variables (see README.md for details)
4. Run the development server:
   ```bash
   npm run dev
   ```

### 3. Find an Issue to Work On

- Browse [open issues](https://github.com/ItsVikasA/Innovision7/issues)
- Look for issues labeled with difficulty levels:
  - `level 1` - Beginner friendly (UI improvements, documentation, minor bug fixes)
  - `level 2` - Intermediate (new features, API integration, component development)
  - `level 3` - Advanced (architecture changes, complex features, performance optimization)
- Comment on the issue to express your interest before starting work

### 4. Create a Branch

Create a new branch for your feature or fix:
```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/bug-description
```

Branch naming conventions:
- `feature/` - New features
- `fix/` - Bug fixes
- `docs/` - Documentation updates
- `refactor/` - Code refactoring
- `test/` - Adding or updating tests

### 5. Make Your Changes

#### Code Style Guidelines

- **JavaScript/React**: Follow existing code patterns
- **Components**: Use functional components with hooks
- **File Structure**: Keep components modular and reusable
- **Comments**: Add comments for complex logic
- **Naming**: Use descriptive variable and function names

#### Best Practices

- Write clean, readable code
- Follow React and Next.js best practices
- Ensure responsive design (mobile, tablet, desktop)
- Test your changes thoroughly
- Keep commits atomic and well-described

### 6. Commit Your Changes

Write clear, concise commit messages:
```bash
git add .
git commit -m "feat: add dark mode toggle to dashboard"
```

Commit message format:
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting, missing semicolons, etc.)
- `refactor:` - Code refactoring
- `test:` - Adding or updating tests
- `chore:` - Maintenance tasks

### 7. Push and Create Pull Request

1. Push your branch:
   ```bash
   git push origin feature/your-feature-name
   ```
2. Go to your fork on GitHub
3. Click "Compare & pull request"
4. Fill out the PR template with:
   - Clear description of changes
   - Related issue number (e.g., "Closes #123")
   - Screenshots/GIFs for UI changes
   - Testing steps

If you find a bug, please create an issue with:

- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Environment details (OS, browser, Node version)
